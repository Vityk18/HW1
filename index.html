Файл .gitignore використовується в системі контролю версій Git для вказівки файлів та папок, які не потрібно включати до репозиторію. Коли Git зустрічає файл .gitignore в кореневій директорії проекту або в будь-якій піддиректорії, він використовує цей файл, щоб визначити, які файли ігнорувати при виконанні операцій додавання, коміту та інших операцій.

git revert - це команда в системі контролю версій Git, яка дозволяє створити новий коміт, який скасовує зміни вказаного коміту або набору комітів. При використанні команди git revert, зміни не видаляються безпосередньо з історії репозиторію, а замість цього створюється новий коміт, який скасовує зміни з попереднього коміту чи комітів.

git reset - це команда в системі контролю версій Git, яка дозволяє переміщати гілку або зміщувати покажчик коміту на певний коміт у репозиторії. Команда git reset може бути використана для скасування змін у коміті, перенесення гілки на інший коміт або перенесення головної гілки (HEAD) на попередній коміт.

git rebase - це команда в системі контролю версій Git, яка дозволяє перенести або перебазувати коміти з одної гілки на іншу гілку. Перебазування комітів застосовується для зміни історії комітів, шляхом застосування змін на іншу базову гілку.

git cherry-pick - це команда в системі контролю версій Git, яка дозволяє вибрати окремі коміти з однієї гілки і застосувати їх до іншої гілки. Команда git cherry-pick дозволяє копіювати вибрані зміни з одного коміту в інший коміт без необхідності переносити всю гілку.

git stash - це команда в системі контролю версій Git, яка дозволяє тимчасово зберегти незакомічені зміни в робочій директорії та стендінг-області (staging area), щоб переключитись на іншу гілку або виконати інші операції, не зберігаючи проміжний коміт.
